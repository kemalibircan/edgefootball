# Football AI - Quick Start Guide

## 🚀 Start the Application

### 1. Backend (Terminal 1)

```bash
# Install dependencies (if not already done)
pip install -r requirements.txt

# Set OpenAI API key
export OPENAI_API_KEY="sk-your-key-here"

# Start FastAPI server
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001
```

### 2. Frontend (Terminal 2)

```bash
cd web
npm run dev
```

**Access:** http://localhost:3001

## 🛠️ Development Tips

### Adding New Translations

1. Edit `web/src/i18n/terms.tr.ts`:
```typescript
export const uiTextTr = {
  myNewSection: {
    title: "Yeni Başlık",
    description: "Açıklama",
  },
};
```

2. Edit `web/src/i18n/terms.en.ts`:
```typescript
export const uiTextEn = {
  myNewSection: {
    title: "New Title",
    description: "Description",
  },
};
```

3. Use in component:
```jsx
const { t } = useLanguage();
<h1>{t.myNewSection.title}</h1>
```

### Creating Theme-Aware Components

```jsx
// Component will automatically adapt to light/dark mode
function MyComponent() {
  return (
    <div className="glass-card">
      <h2 className="text-primary">Title</h2>
      <p className="text-secondary">Description</p>
      <button className="btn-primary">Action</button>
    </div>
  );
}
```

### Using Chat Context

```jsx
import { useChat } from "../contexts/ChatContext";

function MyComponent() {
  const { askFromAction, openSidebar } = useChat();
  
  const handleAskAi = () => {
    askFromAction({
      fixture_id: 12345,
      home_team_name: "Team A",
      away_team_name: "Team B",
      // ... other match data
    });
    openSidebar();
  };
  
  return <button onClick={handleAskAi}>AI'a Sor</button>;
}
```

## 🐛 Common Issues

### Issue: Chat sidebar doesn't open
**Solution:** Check that ChatProvider is in App.jsx and ChatSidebar is in SiteLayout.jsx

### Issue: Theme doesn't persist
**Solution:** Check localStorage for `football_ai_theme` key. Browser may have cookies disabled.

### Issue: DALL-E generation fails
**Solution:** 
1. Verify `OPENAI_API_KEY` in `.env`
2. Check OpenAI account has credits
3. Review backend logs: `ModuleNotFoundError: No module named 'openai'` means run `pip install openai`

### Issue: Images not displaying
**Solution:** Ensure `app/static/slider/` directory exists: `mkdir -p app/static/slider`

### Issue: Language doesn't change
**Solution:** Hard refresh browser (Cmd+Shift+R or Ctrl+Shift+R)

## 💡 Tips & Best Practices

### For Content Admins

1. **Generate slider images weekly** to keep content fresh
2. **Monitor DALL-E costs** - Each image ~$0.04-0.08
3. **Use daily highlights** to surface best matches

### For Developers

1. **Use the contexts** - Don't prop-drill theme/language
2. **Follow CSS variable naming** - Use semantic names
3. **Test both themes** - Dark mode catches contrast issues
4. **Test both languages** - Catches hardcoded text

### For Designers

1. **Color palette is fixed** - Navy + lime only (brand consistency)
2. **Glassmorphism is key** - Use `glass-card` class
3. **Neon glows sparingly** - On key interactive elements
4. **Animations subtle** - 0.2-0.3s transitions
